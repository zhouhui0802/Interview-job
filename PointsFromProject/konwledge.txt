问题1：
SpringBoot项目在maven中进行install的时候报出
Execution repackage of goal org.springframework.boot:spring-boot-maven-plugin:2.3.4.RELEASE:repackage failed:
Unable to find main class 错误

解法1：因为有些模块没有启动类，导致找不到main.class,所以在每个包中添加SpringBoot的启动类

问题2：
配置文件：com.zaxxer.hikari.HikariDataSource飘红

解法2：
在maven中添加包
<dependency>
   <groupId>org.springframework.boot</groupId>
   <artifactId>spring-boot-starter-jdbc</artifactId>
</dependency>

问题3：
解救跨域问题
解法3：
一般来说，在SpringBoot的后台加上@CrossOrigin(但是尚医通那个却一直报错，暂时还没找到解决方案)

知识点：
nginx反向代理
配置内容：
在配置文件里面添加
location ~ /core/ {
	proxy_pass http://localhost:8110;
}
location ~ /sms/ {
	proxy_pass http://localhost:8120;
}
location ~ /oss/ {
    proxy_pass http://localhost:8130;
}

问题4：
nuxt2跟Node.js版本不匹配的问题
解决方案4：
（1）首先安装nvm 这是个Node.js的版本管理工具
（2）nvm list 查看所有已经安装的Node.js的版本
（3）nvm install 20.5.1(Node.js的版本号)
(4) nvm use 20.5.1 //环境中将使用20.5.1的版本

问题5：
报出依赖的库出错
解决方案5：
极有可能是maven下载额依赖不全（比如spring-cloud-starter-openfeign-2.2.5.RELEASE.jar时出错; error in opening zip file错误），
完整的解决方案，去最原始的仓库 https://repo.maven.apache.org/maven2/
找到对应的库，然后跟自己的本地仓库进行对比，看是否有偏差
